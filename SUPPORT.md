# Support

Use this page for non-security help, usage questions, and contributor guidance.

## Before opening a question

1. Read `README.md`.
2. Read `docs/INDEX.md`.
3. Read `CONTRIBUTING.md`.
4. Check open issues for existing discussion.

## Where to ask for help

1. Open a GitHub issue using the templates in `.github/ISSUE_TEMPLATE/`.
2. Use clear repro steps and include the exact command output when reporting problems.
3. For roadmap/scope questions, reference:
   - `docs/governance/SCOPE_GUARDRAILS.md`
   - `REFERENCE_RUNTIME_BOUNDARY.md`

## Security issues

Do not open a public issue for suspected vulnerabilities.

Report security concerns privately per:
- `SECURITY.md`
- `info@faxp.org`

## Maintainer expectations

This is an open-source project with limited maintainer bandwidth.

Best-effort support is provided, but response times vary by:
1. severity
2. reproducibility
3. alignment with scope and roadmap

PRs with tests and clear validation evidence are prioritized.
